git init

Ініціалізація git в обрану директорію

git status

Перегляд статусу змін в файлах

git add

Додавання для відслідкування git файлів. При використанні git add "file_name" - буде додано обраний файл, для додавання всіх файлів необхідно використовувати git add .

git commit

Комміт змін файлів. Для комміту необхідно використовувати команду git commit -m "Description_commit"

git log

Використовується для перегляду коммітів в репозиторії

git checkout

При використанні даної команти у вигляді git checkout "fiel_name" буде повертати його в первинний стан до комміту

git branch

Перегляд гілки на якій розташовані, та перегляд гілок які наявні.
При використанні git branch "name_new_branch" буде сформовану нову гілку.
При використанні git branch -D "name_branch" буде здійснено видалення гілки

git push

Переміщення з локального репозиторію до GitHub певних змін в проекті

git pull

Копіювання останніх змін з серверу репозиторію на локальний GIT

git merge

Поєднання гілок (під master/main). При використання під гілкою мастер ввівши git merge "name_of_branch" буде здійсненно приєднання обраної гілки до гілки master/main

git clone

Копіювання відділеного репозиторію на локальний. Використовується наступним чином git clone "link_repository", після буде виконано еопіювання репозиторію з віддаленого на локальний

Advanced команди GIT

git revert

Повертає деякі існуючі коміти.
При використанні git revert HEAD~3 - Скасує зміни, вказані четвертим останнім комітом у HEAD, і створить новий коміт із скасованими змінами.
При використанні git revert -n master~5..master~2 - Повертає зміни, внесені комітами з п’ятого останнього коміту в master (включено) до третього останнього коміту в master (включено), але не створює жодного коміту зі скасованими змінами. Повернення змінює лише робоче дерево та індекс.

git reset

git reset --hard "commit_name" Вказуємо коміт починаючи з якого необхідно забути всі зміни, при цьому з локальної гілки всі коміти з вказаного будуть видалені

git rebase

Один з методів який дозволяє об'єднати зміни двух гілок. В даному способі є перевага перед merge він дозволяє переписати історію гілки, надав тій історії той вигляд, який необхідний.
Щоб уникнути втрати комітів необхідно перед rebase створити додаткову гілку git branch backup яка буде надавати змогу повернути якщо будуть проблеми зі злиттям

git cherry-pick

Команда за допомогою якої можна вибірково задіяти коміти GIT до поточної робочої гілки HEAD За його допомогою можна вибрати коміт з однієї гілки та задіяти до іншої. Такох являється зручним способом відміни змін. Тобто якщо помилково коміт попав не в ту гілку що необхідно , є змога переключитись на необхідну гілку та перенести куди потрібно даний коміт
Приклад: Необхідно перейти в гілку main та перенести в неї коміт name_commit. Перед цим необхідно переконатись чи дійсно розташовані в необхідній гілці виконавши команду git checkout main після чого необхідно виконати команду git cherry-pick name_commit та необхідний нам коміт буде розташований в кінці гілки main

git stash

Дозволяє на певний час відложити зміни зроблені в робочій копії. ТОбто зберігає не підтверджені зміни в сховищі, для змоги повернення до них пізніше. Для використання необхідно виконати команду git stash.
Щоб повернутись та виконати відложені зміни необхідно виконати команду git stash pop. При вилученні відложених змін вони видаляються з набору та застосовуються до робочої копії.
Для застосування до робочої копії без видалення необхідно використовувати команду git stash apply
